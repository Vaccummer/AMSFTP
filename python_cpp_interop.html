Python 与 C++ 互调用机制

Python 调用 C++ (pybind11)

关键点：不是编译成 C，而是通过 Python C API 动态链接 
Python 代码
    ↓
import AMSFTP  →  加载 .pyd/.so 动态库
    ↓
调用 func()   →  Python 解释器通过函数表找到 C++ 函数指针
    ↓
执行 C++ 原生代码（已编译的机器码）
    ↓
返回值通过 PyObject* 转换回 Python 对象

pybind11 做的事情：

  1. 生成符合 Python C API 规范的包装函数
  2. 注册函数到模块的方法表 (PyMethodDef)
  3. 处理 PyObject* ↔ C++ 类型的转换

C++ 调用 Python

关键点：通过 Python 解释器执行字节码，不是插入 C 源码 
C++ 代码
    ↓
py::gil_scoped_acquire  →  获取 GIL（解释器锁）
    ↓
disconnect_cb(...)      →  调用 PyObject_Call()
    ↓
Python 解释器执行字节码（.pyc）  ← 解释执行，不是插入C源码
    ↓
返回 PyObject*，pybind11 转换为 C++ 类型

性能对比

调用方向 机制 性能
Python → C++ 直接调用编译好的机器码 ✅ 快
C++ → Python 通过解释器执行字节码 ⚠️ 慢（解释执行）

总结

Python 本身不会编译成 C。CPython 解释器是用 C 写的，但它是解释执行 Python 字节码，不是把 Python 翻译成 C。
